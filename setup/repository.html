    Maven Repository

    This page will describe how to configure access to the secure maven repository giving access to various artifacts included in the DexGuard distribution.

    Configuring the secure maven repository / dependencies¶

    For the DexGuard Gradle plugin, you can include the following in your root level Gradle files:

    settings.gradle.kts 

    pluginManagement {
    repositories {
        maven {
            credentials {
                username = "maven"
                password = "${extra["dexguardMavenToken"]}"
            }
            url = uri("https://maven.guardsquare.com")
            // Only search for artifacts with groupId "com.guardsquare.*".
            // Supported since Gradle 5.1.
            content {
                includeGroupByRegex("com\\.guardsquare.*")
            }
            authentication {
                create<BasicAuthentication>("basic")
            }
        }
    }
}
    gradle/libs.versions.toml
    [versions]
dexguard = "9.10.5"

[plugins]
dexguard = { id = "com.guardsquare.dexguard", version.ref = "dexguard" }

            username = "maven"
            password = "${properties["dexguardMavenToken"]}"
        }
        url = uri("https://maven.guardsquare.com")
        // Only search for artifacts with groupId "com.guardsquare.*",
        // supported since gradle 5.1.

            includeGroupByRegex("com\\.guardsquare.*")
        }
        authentication {
            create<BasicAuthentication>("basic")
        }
    }
}

    In either case, you can then include the following in your module level build.gradle(.kts) file:

    build.gradle(.kts)
    dependencies {
    // Example: include the DexGuard runtime library.
    implementation(libs.dexguard.runtime)
}
    gradle/libs.versions.toml
    [versions]
dexguard = "9.10.5"

[libraries]
dexguard-runtime = { group = "com.guardsquare", name = "dexguard-runtime", version.ref = "dexguard" }

    build.gradle(.kts)
    dependencies {
    // Example: include the DexGuard runtime library.
    implementation('com.guardsquare:dexguard-runtime:9.10.5')
}

    and in gradle.properties define the Maven access token:

    dexguardMavenToken = <token>

    The Maven username is always maven. The Maven access token can be obtained as follows:

    1. Log in to the Guardsquare platform.
    2. Go to your Team Settings page using the menu button on the top left.
    3. Go to the "Automation" tab using the button on the left.
    4. At the top of this page, you should see your Maven access token. If you don't already have one, you can generate it by clicking the blue Generate button.
    5. Afterward, you can view your token by clicking the eye icon.

    Don't commit your Maven access token credentials!

    In order to avoid the need to commit the Maven access token credentials to source code repositories, you can define an empty property (dexguardMavenToken=) in your project-specific gradle.properties file and define the actual token in the user specific gradle.properties file located under ~/.gradle. Gradle will replace this property with the one defined in the user properties.
